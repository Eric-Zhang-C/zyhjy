QEMU中的缺页异常处理机制

发表于 2023-12-18 更新于 2023-12-22 阅读次数：
本文字数： 8.8k 阅读时长 ≈ 15 分钟

缺页异常(Page Faults)属于ARM V8处理器的异常类型中的同步异常。当MMU走表时可能会产生若干种类型的MMU faults(有同步的也有异步的)，其中的同步异常，即缺页异常。

本文以QEMU全系统模拟AArch64的场景为例，介绍QEMU是如何处理缺页异常的。

0. 缺页异常的原因

如果虚拟地址是合法的，但MMU在地址翻译的过程中，发生了TLB Miss以及查找页表项失败，那么MMU将会产生缺页异常。

造成缺页异常的原因有三种：

  1. 地址空间映射关系未建立
    • 操作系统还没有对GVA所在的虚拟页分配相应的物理页。如malloc/mmap申请虚拟的地址空间并未分配实际物理页，首次访问触发缺页异常。
  2. 地址空间映射关系已建立
    • 要访问的虚拟页所对应的物理页已经从内存中被换出到磁盘中
  3. 访问的地址空间不合法
    • 非法操作访问越界、空指针解引用等都会出现缺页异常

1. 缺页异常的产生

在QEMU全系统模拟下模拟Guest程序执行时，需要将Guest Virtual Address(GVA)翻译成Guest Physical Address(GPA)，这一过程对应的函数为target/arm/helper.c::get_phys_addr。

在进行地址翻译时，MMU首先在TLB中查找是否存在GVA对应的entry，如有则地址翻译结束--这一过程被称为Fast Path。 如果TLB未命中，则会进行pagetable walk，即在页表中查询GVA对应的GPA--由于多级页表的访问往往伴随着多次的内存读取请求，页表查询的速度较TLB而言慢很多，因此这一过程被称为Slow Path。

LPAE（Large Physical Address Extension）是ARM架构中的一种特性，用于支持更大的物理地址范围。

在LPAE中，页表结构被扩展为多级结构，其中每一级都包含更多的页表项，用于映射更大的地址范围。特别是，LPAE中引入了64位的页表项，称为LPAE页表项（LPAE Page Table Entry）。这些页表项包含更多的位用于存储物理地址和其他控制信息。

实际的Slow Path过程在函数target/arm/helper.c::get_phys_addr_lpae中完成。如果产生ARMFault_Translation，则说明出现缺页异常，相关错误信息记录在结构体ARMMMUFaultInfo中。

2. 缺页异常的传递

在target/arm/helper.c::do_ats_write中

根据地址翻译过程产生的错误信息为ESR(Error Syndrome Register)寄存器赋值，并抛出异常。

在AArch64体系结构上，缺页异常并没有一个专门的异常号，而是与其他一些用户态同步异常共用一个异常号，即8号同步异常，操作系统需要根据ESR(Error Syndrome Register)寄存器中存储的信息来判断发生的异常是否为缺页异常，如果是，则从FAR_EL1寄存器中取出发生缺页异常时访问的虚拟地址。(在x86-64体系结构上，缺页异常会触发13号异常(#PF)，并且访问出错的虚拟地址会被放在CR2寄存器中)

函数arm_cpu_do_interrupt定义在target/arm/helper.c，具体的操作则会转发给arm_cpu_do_interrupt_aarch64完成，下面是该函数的主要工作：

  1. 从env中获取目标异常级别new_el以及对应级别的向量基地址addr。
  2. 根据目标异常级别和当前异常级别，确定是否需要切换到AArch64模式以及对应的向量偏移地址。根据不同的异常类型，对异常发生时的寄存器状态进行处理，包括设置FAR寄存器、修改syndrome寄存器、保存/恢复旧模式等。
  3. 根据异常类型和目标异常级别，计算最终的向量地址addr。
  4. 保存旧模式到对应的堆栈指针状态寄存器。根据处理后的异常级别和模式，更新PSTATE的值，设置AArch64模式标志，并恢复新EL的堆栈指针。
  5. 将PC设置为计算得出的向量地址。

总体而言，这段代码根据不同的异常类型和异常级别，进行了一系列的寄存器状态处理和模式切换，最终将PC设置为目标异常级别对应的缺页异常函数入口地址。

PSTATE（Processor State）是ARM体系结构中的一个寄存器，用于存储和控制处理器的状态信息。PSTATE寄存器的位字段包含了处理器的执行状态、操作模式、中断使能、指令标志等重要信息。具体来说，PSTATE寄存器包含以下位字段：

  • N（Negative）：表示最近一条指令的结果是否为负数。
  • Z（Zero）：表示最近一条指令的结果是否为零。
  • C（Carry）：表示最近一条指令的进位/借位结果。
  • V（Overflow）：表示最近一条指令的溢出结果。
  • EL（Exception Level）：表示当前的异常级别。
  • SP（Stack Pointer）：表示当前使用的堆栈指针模式，用于选择使用哪个堆栈指针寄存器。
  • DAIF（Disable Asynchronous Interrupts Flags）：控制异步中断的使能和禁用。
  • PAN（Privileged Access Never）：用于控制特权访问的限制。

通过修改PSTATE寄存器的不同位字段，可以控制处理器的执行状态、中断处理、异常处理、特权级别等。在异常处理过程中，PSTATE寄存器的值会被保存和恢复，以确保异常处理的正确性和一致性。。
