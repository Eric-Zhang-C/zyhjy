1. OSDI七层模型

1. 七层模型和四层模型

OSDI七层模型 Internet协议套件--TCP/IP协议栈 数据单位
Application Layer（应用层） 提供用户应用程序接口，包括文件系统、网络协议栈、图形界面等 提供应用程序间的通信服务和协议，例如HTTP、FTP、SMTP、DNS等
Presentation Layer（表示层） 处理数据的表示和转换，例如数据加密、压缩和格式转换
Session Layer（会话层） 管理通信会话和跨进程的数据交换，包括会话建立、维护和终止
Transport Layer（传输层） 负责数据传输的可靠性和流量控制，提供端到端的数据传输服务 负责提供端到端的数据传输服务，例如TCP和UDP协议 TCP报文段/UDP用户数据包（Segments）
Network Layer（网络层） 处理网络地址和路由选择，实现数据在网络中的传输 处理数据在网络中的路由和转发，例如IP协议 包（Packages）
Data Link Layer（数据链路层） 处理物理层面上的数据传输，负责数据帧的发送和接收 负责处理物理层面上的数据传输，将数据分割成帧并进行传输，例如以太网协议 帧（Frames）
Physical Layer（物理层） 处理实际的物理传输介质，例如电缆、光纤等 比特流（Bits）

2. HTTP

1. 一次完整的HTTP请求过程

  1. DNS解析：在发起HTTP请求之前，客户端需要将目标主机名解析为对应的IP地址。客户端会向DNS服务器发送域名解析请求，获取目标主机的IP地址。
  2. 建立TCP连接：客户端使用解析得到的目标主机IP地址，通过TCP/IP协议与目标服务器建立连接。这个过程通常使用三次握手来确保连接的可靠性和稳定性。
  3. 发起HTTP请求：一旦TCP连接建立，客户端会发送完整的HTTP请求给服务器。请求包括请求行、请求头部和请求体，其中请求行指定请求的方法、URL和协议版本。
  4. 服务器处理请求：服务器接收到客户端发送的HTTP请求后，会根据请求的内容进行处理。这可能包括读取请求头部信息、解析请求体、执行请求的处理逻辑等。
  5. 服务器返回HTTP响应：服务器处理完请求后，会生成对应的HTTP响应。响应包括响应状态行、响应头部和响应体。响应状态行指定响应的状态码和对应的文本描述。
  6. 接收HTTP响应：客户端接收到服务器发送的HTTP响应后，会读取响应的内容。这可能包括读取响应头部信息和响应体内容。
  7. 关闭TCP连接：在完成HTTP响应的接收后，客户端和服务器之间的TCP连接可以关闭。这个过程通常使用四次挥手来确保连接的正确关闭。

2. GET和POST的区别：

  • GET是获取数据，POST是修改数据。

  • 数据位置：GET请求通过URL的查询参数传递数据，即将数据附加在URL的末尾，例如：http://example.com/page?param1=value1&param2=value2。而POST请求将数据放在请求体中进行传输，不在URL中暴露。

  • 数据长度限制：由于GET请求的数据是通过URL传递，URL长度有限制，不同浏览器和服务器对URL长度的限制不同，一般在几千个字符左右。而POST请求的数据放在请求体中，没有明确的长度限制，但服务器可能会设置对请求体的大小进行限制。

  • 安全性：GET请求的数据在URL中可见，容易被拦截、截获或缓存，包含敏感信息的话可能存在安全风险，例如密码。POST请求的数据在请求体中，相对于GET请求更加安全，因为请求体中的数据不会出现在URL中。

  • 缓存机制：GET请求可以被浏览器缓存，当再次请求相同URL时，浏览器可以直接从缓存中获取响应结果。POST请求默认不会被缓存，每次请求都会向服务器发送请求。

  • 幂等性：GET请求是幂等的，即多次重复请求不会产生副作用，只是获取相同的资源。而POST请求通常用于发送数据并对服务器资源进行修改，多次发送同样的POST请求可能会产生不同的结果。

  • 使用场景：GET请求适合获取资源、查询数据等无副作用的操作，例如浏览网页、搜索等。POST请求适合发送数据、创建资源、更新资源等带有副作用的操作，例如提交表单、上传文件等。
